# Example Catalog

A minimal example showing how to structure catalog documentation. Contains a simple customer/orders schema to demonstrate the documentation format.

- `index.md` — Entry point linking to table docs, concepts, and examples.
- `tables/` — Table schema documentation (columns, types, relationships).
- `concepts/` — Maps natural language terms to SQL patterns (e.g., "best" to `ORDER BY ... DESC`).
- `examples/` — Example SQL queries the LLM can reference.

Use this as a template when creating documentation for your own catalogs.
